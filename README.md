# Sireh AI Engine Backend

Backend foundation for the Sireh AI Commerce Platform.

## Overview

Sireh AI Engine Backend provides the server-side foundation for commerce automation,
AI-assisted workflows, and integration services used by the Sireh AI Commerce Platform.
The backend is designed for a production-grade Node.js environment with strict TypeScript,
modern ESM modules, and tooling that remains compatible with Shopify Remix app development.

## Tech Stack

- Node.js 22
- TypeScript 5.x
- ESM modules
- ESLint 9 flat config
- Prettier 3.x
- Shopify Remix App compatible tooling
- Express
- Prisma
- PostgreSQL-ready data layer
- OpenAI SDK integration

## Folder Purpose

- `src/`: Backend application source code.
- `src/config/`: Runtime configuration and environment handling.
- `src/middleware/`: HTTP middleware such as security, logging, and request handling.
- `src/routes/`: API route definitions.
- `src/services/`: Business logic and external service integrations.
- `src/utils/`: Shared utility functions.
- `dist/`: Compiled production output generated by TypeScript.
- `coverage/`: Test coverage output.

## Available Commands

- `npm run dev`: Start the backend in development watch mode.
- `npm run build`: Compile TypeScript output into `dist`.
- `npm run start`: Run the compiled production server from `dist`.
- `npm run typecheck`: Run TypeScript checks without emitting files.
- `npm run lint`: Run ESLint with zero warnings allowed.
- `npm run lint:fix`: Automatically fix supported ESLint issues.
- `npm run format`: Format files with Prettier.
- `npm run format:check`: Check formatting without writing changes.
- `npm run test`: Run the test suite.
- `npm run test:watch`: Run tests in watch mode.
- `npm run test:coverage`: Run tests with coverage reporting.
- `npm run validate`: Run typecheck, lint, format check, and tests.
- `npm run clean`: Remove generated build and coverage output.

## Environment Variables

Create a local `.env` file for development values. Do not commit secrets.

```ini
NODE_ENV=development
PORT=3000
DATABASE_URL=
OPENAI_API_KEY=
SHOPIFY_API_KEY=
SHOPIFY_API_SECRET=
SHOPIFY_APP_URL=
```

Use `.env.example` for safe placeholder documentation when the project adds a shared
environment template.

## Development Workflow

1. Install dependencies with `npm install`.
2. Create a local `.env` file with required development values.
3. Run `npm run dev` for local development.
4. Keep code inside `src/` and use configured path aliases where appropriate.
5. Run `npm run lint` and `npm run typecheck` before handing off changes.
6. Use `npm run format` or editor Prettier integration to keep formatting consistent.

## Verification Workflow

Use the following checks before production handoff:

```bash
npm run typecheck
npm run lint
npm run format:check
npm run test
npm run build
```

For a full local quality gate, run:

```bash
npm run validate
```
